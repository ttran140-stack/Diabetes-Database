# Diabetes Database
The creation process of a functional database from a real-world health dataset. Proposes reports that documents the iterations and development of a scalable schema with 10 entities that analyzes patient information such as lifestyle factors, genetic information, medical records, and organ functions.

The goal of this project was to apply database design principles, normalize, and use SQL querying skills to create an efficient schema and functional database for storing healthcare data while supporting meaningful analysis and reporting.

# Repository Structure
* **Project Proposal INST327.pdf** - A proposal report that describes a description of the dataset and its different research purposes, the audience of the dataset, the first iterations of entities for the schema for the database, and general ethical considerations
* **Diabetes Database Progress Report.pdf** - A progress report that presents the first iteration of the schema with an image of an ERD. Also present how sample data is taken from the dataset, changes from the proposal, progress made since the proposal, various ethical considerations, and plans for the remaining work for the project.
